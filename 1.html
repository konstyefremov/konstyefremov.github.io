python packages

A package in Python is a program which you can use by itself, or which you can reuse in your project. There are lots of packages for different purposes. To perform mathematical operations, to work with databases, to built websites etc. You can even find a package which downloads videos from Youtube.

Noncomplete list of packages can be found at The Python Package Index. At the moment (March 2019) there are 173,483 projects listed on the website. So, it's obvious why Python comes pre-installed with only the most essential packages.

pip

Python comes with a command line utility pip to manage packages. With it you can see which packages are available on your system, install new ones, remove not needed anymore.

  • To install a package. Open a command line and write pip install <package-name>. For example, a command pip install flask will install the latest version of Flask. You also can install a specific version of package. For example, a command pip install flask==0.12.4 will install Flask version 0.12.4.
  • To see which packages are available on your system, write pip list
  • To remove a package, write pip remove <package-name>

virtual environments

Most of the programmers build and maintain many projects. At the same time packages evolve. They remove some features, add anothers.

Now imagine (a typical) situation. A programmer built a project and shipped it to the client 2 years ago with some package. Since then, this package changed and his project became incompatible with the new version of the package. One day the client calls and asks to fix some problem with the project. If the programmer will install a new version of the package, he will need to do lots of work to make the project compatible with the new version of the package. But he needs to fix only some small problem. If he will install two versions of this package on his computer, the new will override the old one.

For this problem (and some others), python provides a tool called virtual environment or venv. With this tool (actually a package) you can create separate environments for each project and support them independently from each other.

To get started, you will need:

  • install virtual environment package on your computer pip install virtualenv
  • install the package to make it easier to work with on Windows pip install virtualenvwrapper-win
  • create a new virtual environment, write mkvirtualenv <folder name>
    This command will copy python executables to the folder and enter the new environment. You will see the name of environment in brackets before the name of the current folder.
  • to activate the environment write <folder-name>\Scripts\activate.bat
  • to deactivate write deactivate
